# vSelfTraining — 半監督式自訓練模型報告

## 任務說明

在 V1（TF-IDF + Logistic Regression）基礎上，採用 **Self-Training（自訓練）** 半監督學習方法，同時利用已標記的 `train_2022.csv` 與未標記的 `test_no_answer_2022.csv` 來訓練模型，預測結果輸出至 `vSelfTraining.csv`。

## 資料概述

| 項目 | 說明 |
|------|------|
| 已標記資料 | `train_2022.csv`，2,000 筆，含 `row_id`、`TEXT`、`LABEL` |
| 未標記資料 | `test_no_answer_2022.csv`，11,000 筆，含 `row_id`、`TEXT` |
| 標籤分布 | 正面 (1) 與負面 (0) 各 1,000 筆，完全均衡 |

## 方法論 — Self-Training（自訓練）

### 核心概念

Self-Training 是一種經典的半監督學習方法。其核心思想是：

1. 先用已標記資料訓練初始模型
2. 用該模型對未標記資料進行預測
3. 將 **高信心度** 的預測結果作為「偽標籤」（pseudo-labels）加入訓練集
4. 重新訓練模型
5. 重複步驟 2–4 直至收斂

### 為何適用此任務

- **已標記資料少**（僅 2,000 筆），未標記資料多（11,000 筆），符合半監督學習場景
- 透過偽標籤擴增訓練集，模型可學習到更豐富的語言特徵
- TF-IDF 在全部文本上 fit，利用未標記資料改善特徵表示的覆蓋率

## 模型架構

### 特徵提取 — TF-IDF（Transductive）

- 最大特徵數：10,000
- N-gram 範圍：(1, 2)（unigram + bigram）
- 使用 sublinear TF（對詞頻取 log）
- **關鍵差異**：TF-IDF 在所有文本（train + test）上 fit，確保特徵空間覆蓋更完整

### 分類器 — Logistic Regression

- 正則化參數 C = 1.0
- 最大迭代次數：1,000

### Self-Training 超參數

| 參數 | 設定值 | 說明 |
|------|--------|------|
| 初始信心度閾值 | 0.85 | 第一輪使用的信心度門檻 |
| 最低信心度閾值 | 0.75 | 閾值遞減的下限 |
| 閾值遞減量 | 0.02 | 每輪迭代降低的閾值 |
| 每輪最大新增量 | 500 | 限制每輪加入的偽標籤數量，防止噪音過多 |
| 最大迭代次數 | 15 | 防止無限迭代 |

### 遞減閾值策略

V1 基線模型準確率僅約 66%，直接使用 0.95 的高閾值會導致**完全無法產生偽標籤**。因此採用：

1. **初始閾值 0.85**：從較寬鬆的門檻開始
2. **逐輪遞減 0.02**：隨著模型因偽標籤而增強，逐步放寬門檻
3. **最低 0.75**：避免閾值過低引入過多噪音
4. **批次限制 500**：每輪最多加入 500 筆最高信心度的偽標籤，確保品質

## 訓練流程

```
載入資料
  ↓
在所有文本上 fit TF-IDF（transductive）
  ↓
基線模型 5-fold CV（對照 V1）
  ↓
┌─────────── Self-Training Loop ───────────┐
│ 1. 用已標記 + 偽標籤資料訓練模型           │
│ 2. 對剩餘未標記資料預測                     │
│ 3. 篩選信心度 ≥ 閾值的樣本                 │
│    - 若無 → 降低閾值並重試                  │
│ 4. 取最高信心度的前 500 個加入偽標籤        │
│ 5. 閾值遞減 0.02（下限 0.75）              │
│ 6. 若所有未標記資料已納入 → 停止            │
└──────────────────────────────────────────┘
  ↓
在全部訓練資料（原始 + 偽標籤）上訓練最終模型
  ↓
對整個測試集做最終預測
  ↓
輸出 vSelfTraining.csv
```

## 與 V1 的差異

| 項目 | V1 | vSelfTraining |
|------|-----|---------------|
| 學習方式 | 監督式 | 半監督式（Self-Training） |
| TF-IDF fit 資料 | 僅 train | train + test（transductive） |
| 訓練資料 | 2,000 筆（僅已標記） | 2,000 + 偽標籤（動態擴增） |
| 閾值策略 | — | 遞減閾值（0.85 → 0.75） |
| 核心改進 | — | 利用未標記資料的資訊提升模型泛化能力 |

## 風險與注意事項

1. **錯誤累積**：若初始模型品質不佳，錯誤的偽標籤會被傳播並放大
2. **閾值敏感**：信心度閾值過低會引入噪音，過高則偽標籤數量不足
3. **類別不平衡**：Self-Training 可能偏向某一類別的偽標籤，導致分布偏移
4. **設計緩解措施**：
   - 使用遞減閾值策略，兼顧品質與數量
   - 每輪限制最多加入 500 筆偽標籤
   - 監控每輪迭代的新增數量、信心度與類別分布
